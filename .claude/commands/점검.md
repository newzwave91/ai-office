지식금고 위키(vault/00_지식금고/)를 **2겹으로** 점검(lint)하라 — 기계는 형식을, AI는 의미를 본다.

## 1겹 — 기계 검사 (형식: 스크립트가 규칙으로)

먼저 결정적 린트 스크립트를 실행해 **형식 오류**를 받아온다. 이쪽은 추측 없이 규칙으로 잡는다.

```
node .claude/scripts/lint-vault.mjs vault
```

- 출력에 들어오는 것: **깨진 위키링크**(대상 노트 없음), **고아 노트**(들어오는 링크 0), **인덱스 정합성**(index.md·스킬 인덱스의 유령 항목/미등록 노트).
- 스크립트가 실행되면 그 결과를 형식 검사 결과로 그대로 신뢰한다. AI가 같은 항목을 중복해서 다시 읽지 마라(토큰 낭비).
- **node가 없거나 스크립트가 실패하면** 그때만 위 3개 형식 항목(깨진 링크/고아/인덱스 불일치)을 AI가 직접 읽어 점검한다(폴백).

## 2겹 — AI 검사 (의미: 규칙으로 못 잡는 것)

기계가 못 보는 **내용**을 AI가 위키를 직접 읽고 판단한다.

1. **모순**: 노트 간 서로 충돌하는 주장. 어느 쪽이 최신·정확인지 판단 근거와 함께 보고.
2. **중복**: 같은 사실을 다루는 노트가 분산 — 통합 후보 제시.
3. **낡은 정보**: 날짜가 오래됐거나 이후 노트·원문과 어긋나는 주장.
4. **빈 구멍**: 자주 언급되는데 자기 노트가 없는 개념, 채울 가치가 있는 데이터 공백 — 다음에 조사할 질문 제안.
5. **규모**: index.md의 한 분야 항목이 과다(예: 분야당 30개+)하면, 그 분야를 하위 분류로 쪼개고 index.md도 분할할 것을 제안한다. (이 위키는 벡터DB 없는 평문 위키라 규모가 커지면 관련 노트를 놓치기 시작한다.)

## 처리 규칙

- 기계적으로 안전한 수정(인덱스 불일치 동기화, 깨진 링크 교정)은 바로 고친다 — 단 어느 노트로 고칠지는 AI가 판단한다.
- 내용 판단이 필요한 것(모순 해소, 노트 통합·삭제, 분야 분할)은 고치지 말고 보고만 한다.
- 점검 결과를 vault/00_지식금고/log.md에 "## [YYYY-MM-DD] 점검 | 발견 N건" 으로 append한다.
- 보고 형식: 기계 검사 결과 → AI 검사 결과 → 바로 고친 것 → 사용자 결정 필요한 것 → 제안 질문.
