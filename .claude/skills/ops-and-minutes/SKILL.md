# 회의록·업무 기록 절차

회의록, 녹취 정리, 업무보고, 프로세스 문서화 작업은 아래 순서를 따른다.

## 절차

1. **원문 정제**: 입력 원문에서 잡담, 인사말, 반복 발언을 제거한다. 단, 결정사항·액션 아이템과 관련된 내용은 절대 빠뜨리지 않는다.
2. **표준 구조 변환**: 회의록은 다음 구조로 변환한다 — 제목/일시/참석자 → 핵심 결정사항(3~5개) → 논의 요지 → 액션 아이템(담당·기한 명시). 업무보고는 한 일→결과→다음 할 일→막힌 것 순서.
3. **불명확 표시**: 발언자가 불명확하면 추측해서 귀속시키지 말고 [확인 필요]로 표시한다. 원문에 없는 결정사항을 만들어 넣지 않는다.
4. **저장과 연결**: 회의록은 vault/10_회의록/에 "YYYY-MM-DD 제목.md" 형식으로 저장한다. 회의에서 나온 고객 사실·결정사항은 지식 금고의 관련 노트에 [[위키링크]]로 연결하고, 보존 가치가 있으면 해당 분야 폴더에 노트로 기록한다.
