# AI 팀장 오피스 — 설치 · 실행 · 배포 가이드

이 문서는 두 독자를 위한 것입니다.
- **A. 설치하는 사람(대표님)** — 고객 PC에 제품을 깔고 인증까지 마치는 절차
- **B. 빌드·배포하는 사람(대표님)** — exe를 만들어 GitHub에 올리는 절차

---

## A. 고객 PC에 설치하기

### 시스템 요구사항
- Windows 10/11 (64bit)
- 인터넷 연결
- 고객 본인의 **Claude 구독 계정** (Team 요금제 — Standard/Premium seat 모두 Claude Code 포함)

> Node.js는 필요 없습니다. exe 안에 런타임이 내장돼 있고, Claude Code는 첫 실행 때 자동 설치됩니다.

### 설치 절차
1. **다운로드** — GitHub Releases 배포 페이지에서 `AI-Office-Setup-x.x.x.exe`를 받습니다.
2. **설치** — exe를 실행합니다. 미서명이라 Windows SmartScreen 경고가 뜨면 *추가 정보 → 실행*을 누릅니다. 설치 마법사에서 위치를 고르고 진행하면 바탕화면에 **AI팀장오피스** 아이콘이 생깁니다.
3. **첫 실행 (설정 창)** — 아이콘을 더블클릭하면 설정 창이 4단계로 안내합니다.
   - **1단계 · Claude Code 확인/설치** — 없으면 "설치하기" 버튼 한 번으로 자동 설치(수 분 소요, Node 불필요).
   - **2단계 · Claude 계정 로그인** — "로그인 창 열기"를 누르면 터미널이 열리고 브라우저로 OAuth 로그인합니다. 고객의 Claude 계정으로 로그인하세요. 완료되면 자동 감지됩니다.
   - **3단계 · 회사 정보 입력** — 회사명·업종·주요 고객·역할을 입력하고 "저장하고 계속". 이 정보가 AI 팀장들의 회사 컨텍스트가 됩니다. (앱을 다시 실행하면 이 단계에서 언제든 수정 가능)
   - **4단계 · 오피스 실행** — "오피스 실행"을 누르면 본 화면이 뜹니다. 이후 실행마다 1·2단계는 자동 확인되고, 회사 정보를 그대로 두면 바로 진행합니다.

### 사용
- 팀장(마케팅 Echo / 전략 Oracle / 운영 Link / 검토 Glitch / 총괄 Neo)에게 질문하면 Claude가 해당 페르소나로 답합니다.
- 보존 가치가 있는 사실·결정은 지식금고(vault)에 자동 정리됩니다.
- 데이터는 고객 PC에만 저장됩니다(`%APPDATA%/AI팀장오피스/workspace`).

### 음성모드 (선택 · 핸즈프리 음성 입력)
팀장 대화창 입력줄 왼쪽 🎤 버튼이 **음성모드 토글**입니다.
- **켜기(🎙️)** — 말하면 자동으로 인식해 전송합니다. 말 끝나고 ~1초 멈추면 그 구간만 잘라 변환 → 자동 전송(마우스로 다시 누를 필요 없음). 팀장이 답하는 동안엔 듣지 않고, 끝나면 자동으로 다시 듣습니다.
- **끄기(🎤)** — 텍스트 입력만 사용합니다.

음성 인식은 **RTZR(리턴제로) STT**를 사용하며, 쓰려면 키를 한 번 넣어야 합니다(안 넣으면 음성모드만 비활성, 나머지 기능은 정상).
1. https://developers.rtzr.ai/ 가입 → 콘솔에서 `client_id`, `client_secret` 발급
2. `%APPDATA%\AI팀장오피스\stt-config.json` 파일을 만들어 아래처럼 저장:
   ```json
   { "clientId": "발급받은_client_id", "clientSecret": "발급받은_client_secret" }
   ```
   (탐색기 주소창에 `%APPDATA%\AI팀장오피스` 를 입력하면 폴더가 열립니다. 이 파일은 GitHub로 동기화되지 않아 안전합니다.)
3. 앱을 재시작하면 음성모드가 활성화됩니다.

> 과금은 **실제로 말한 음성 길이만큼만** 됩니다(켜두기만 한 침묵·대기 시간은 0원). 가입 시 약 10시간 무료, 이후 종량제. 한국어 받아쓰기에 최적화돼 있습니다.

### 설치 후 작동 확인 (테스트)
설치가 끝나면 아래로 실제 작동을 확인합니다.
- [ ] 바탕화면 아이콘 더블클릭 → 설정 4단계를 마치면 메인 오피스 화면이 열린다
- [ ] 팀장에게 한 줄 질문(예: "이번 주 우선순위 정리해줘") → 잠시 후 **AI가 답하면 핵심 작동 OK** (Claude 호출 성공)
- [ ] CEO 오케스트레이터에 지시 → 담당 팀장에게 분류·배정되고 작업 트래커에 등록된다
- [ ] 🧠 금고 모드로 물은 사실이 좌측 탐색기/지식금고에 노트로 쌓인다
- [ ] 데이터가 `%APPDATA%\AI팀장오피스\workspace`에 저장된다 (개발 폴더와 분리)
- [ ] (선택) RTZR 키 설정 후 음성모드 🎙️ 켜고 말하면 텍스트로 자동 입력·전송된다

> AI가 답하지 않으면 대개 Claude 로그인 문제입니다 — 아래 "문제 해결" 표를 참고하세요.

### 문제 해결
| 증상 | 해결 |
|---|---|
| "Claude Code 실행을 확인하지 못함" | 설정 창을 닫고 다시 실행. 그래도 안 되면 터미널에서 `irm https://claude.ai/install.ps1 \| iex` 수동 실행 후 재시작 |
| 로그인이 감지 안 됨 | 터미널에서 `claude auth status`로 `loggedIn: true` 확인. 아니면 `claude auth login` 재시도 |
| 팀장 응답이 느리거나 한도 초과 | seat 등급 문제. 아래 "모델 조정" 참고 |

### 모델 조정 (한도 관리)
기본 팀장 모델은 **Sonnet**(균형 — 빠르고 똑똑)입니다. 앱 안의 **에이전트 설정**에서 팀장별로 모델을 바꿀 수 있어요. **Premium seat**(Pro의 6.25배 한도)는 전략·창작이 많은 팀장을 `Opus`로 올려도 여유가 있고, **Standard seat**(1.25배)는 Sonnet 유지 또는 단순 업무 팀장을 `Haiku`로 낮추면 한도가 오래갑니다.

## B. exe 빌드 · 배포 (대표님)

### 빌드 환경 (개발 PC)
- Node.js LTS 설치
- 저장소 클론 후 의존성 설치:
  ```powershell
  npm install
  npm install --prefix web
  ```

### 개발 중 실행 (Electron, 빌드 없이 확인)
```powershell
npm run electron
```
→ `web/dist` 빌드 + `seed` 생성 후 Electron 앱이 뜹니다.

### 배포용 exe 만들기
```powershell
npm run dist
```
→ `release/` 폴더에 `AI팀장오피스 Setup x.x.x.exe`(NSIS 설치 파일)가 생성됩니다. 배포할 땐 공백·한글 없는 이름 `AI-Office-Setup-x.x.x.exe`로 올립니다.

빌드 과정이 하는 일:
1. `build:web` — React 프론트를 `web/dist`로 빌드
2. `build:seed` — `.claude` + `CLAUDE.md` + **빈 vault 템플릿**을 `electron/seed`로 생성 (회사 실데이터는 포함 안 됨)
3. `electron-builder` — 위 산출물을 묶어 설치 exe 생성

### GitHub Releases로 배포
1. 제품 저장소(예: `ai-office-product`)에 코드를 push
2. Releases → "Draft a new release" → 태그(예: `v1.0.0`) 생성
3. `AI-Office-Setup-x.x.x.exe`를 첨부(Attach binaries)
4. Publish → 고객에게 그 Release 페이지 링크 전달 → 고객이 exe 다운로드

> ⚠️ **배포본에는 회사 기밀 vault가 절대 들어가면 안 됩니다.** 빌드는 빈 템플릿만 동봉하도록 설계돼 있고, 대표님 실데이터는 별도 private 저장소에만 보관됩니다.

### 아이콘 (선택)
`electron/assets/icon.ico`를 추가하고 `package.json`의 `build.win`에 `"icon": "electron/assets/icon.ico"`를 넣으면 커스텀 아이콘이 적용됩니다.
