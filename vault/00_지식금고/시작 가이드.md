# 🚀 시작 가이드 — 처음 세팅 & 사용법

이 보관소(vault)는 회사의 **AI 세컨드 브레인**입니다. 세 가지 형식이 한 폴더 안에 겹쳐 있습니다.

| 형식 | 무엇 | 어디 |
|---|---|---|
| 📚 **LLM 위키** | *사실* — 무엇이 참인가 (고객·시장·규칙) | `00_지식금고/{분야}/` |
| 🛠 **카파시 스킬** | *노하우* — 무엇이 통하는가 (재사용 패턴) | `00_지식금고/스킬/` |
| 🔗 **옵시디언** | 위 둘을 *연결·항해* (링크·그래프·MOC) | 형식 전체 |

> 핵심 한 줄: **사실은 분야 폴더에, 노하우는 스킬 폴더에.** 둘을 섞지 않는 것이 이 시스템의 뼈대입니다.

---

## 1단계 — 회사 정보 입력 (1분)
앱 **설정**에서 회사명·업종·고객·역할을 입력합니다. 그러면 `CLAUDE.md`의 회사 컨텍스트와 `공통/회사정보.md`가 자동으로 채워지고, 모든 팀장이 그 맥락 위에서 답합니다. (입력 전에는 "(설정에서 입력하세요)" 상태)

## 2단계 — 폴더 구성 이해하기
처음엔 골격만 있고 **비어 있습니다.** 쓰면서 채워집니다.

```
🏠 홈.md                  ← 옵시디언 첫 화면(MOC). 여기서 출발
00_지식금고/              ← AI가 관리하는 위키 (핵심)
├─ index.md               사실 카탈로그
├─ 스킬/스킬 인덱스.md     노하우 카탈로그
├─ 공통·마케팅·전략·운영·검토·총괄/   분야별 사실 노트
├─ log.md / 가이드라인.md  활동기록 / 사용 치트시트
01_원문/ · 02_외부자료/    외부 원본 ⛔수정·삭제 금지(읽기만)
10_회의록/ · 90_산출물/    회의 메모 / 완성 문서
_템플릿/                  새 노트 양식(옵시디언 전용)
```
- **분야 폴더**는 직접 만들지 않아도 됩니다 — 이미 6개가 준비돼 있습니다.
- 외부에서 받은 자료는 `02_외부자료/`에, 완성된 기획서·문서는 `90_산출물/`에 둡니다.

## 3단계 — 세 형식 쓰는 법

### 📚 LLM 위키 (사실 쌓기)
1. 앱에서 팀장에게 **🧠 금고 모드**로 질문합니다.
2. 보존 가치가 있는 사실이 분야 폴더에 노트로 **자동 적립**됩니다. (빠른 모드는 읽기 전용·적립 안 함)
3. `한 노트 = 한 사실, 3문장 이내`. 관련 개념은 **이중 대괄호 위키링크**로 잇습니다.
4. 회사가 따를 결정은 `공통/결정로그.md`에 한 줄씩 쌓여 회사의 기억이 됩니다. → [[결정로그]]

### 🛠 카파시 스킬 (노하우 쌓기)
1. 산출물(기획서·문서)이 하나 나오면, `/스킬` 명령으로 **재사용 가능한 패턴만** 골라 증류합니다.
2. `00_지식금고/스킬/스킬 - 제목.md`로 저장됩니다 (핵심 패턴·단계·체크리스트·예시).
3. 다음에 비슷한 일을 할 때, 작업 전에 [[스킬 인덱스]]에서 관련 스킬을 먼저 읽고 적용합니다.
4. **사실(자동 적립)과 노하우(직접 큐레이션)는 다릅니다** — 노하우는 검증된 것만, 손으로 고릅니다.

### 🔗 옵시디언 (시각적으로 연결)
1. [옵시디언](https://obsidian.md) 설치 → "폴더를 보관소로 열기"로 이 `vault` 폴더를 엽니다.
2. **그래프 뷰**: [[🏠 홈]]을 허브로 지식·스킬이 분야별 색으로 연결돼 보입니다.
3. **백링크**: 한 노트를 누가 참조하는지 역추적됩니다.
4. **템플릿**: 새 노트는 `_템플릿/` 양식을 불러오면 형식(날짜·출처·태그)이 자동으로 맞습니다.
5. 앱만 써도 충분합니다 — 옵시디언은 "더 보고 싶을 때"의 선택지입니다.

## 4단계 — 첫 주 추천 루틴
1. **설정**에서 회사 정보 입력.
2. 팀장과 🧠 금고 모드로 대화 3~5회 → 분야 폴더에 사실이 쌓이는지 확인.
3. 첫 산출물이 나오면 `/스킬`로 스킬 노트 1개 만들기.
4. 주 1회 `/점검`으로 깨진 링크·중복·낡은 노트 정리.

## ⌨️ 명령어 치트시트
- `/수집` 외부자료를 지식금고에 증류 · `/스킬` 산출물→재사용 스킬 노트
- `/기록` 사실 적립 · `/회의` 팀장 회의 · `/점검` 위키 정리(lint)
- 더 짧은 사용 요약은 [[가이드라인]].
